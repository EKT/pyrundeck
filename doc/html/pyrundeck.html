pyrundeck.api module¶

This module contains the core of the API client.

The general idea is to abstract away all possible requests into three methods (RundeckApiClient.get, RundeckApiClient.post and RundeckApiClient.delete), that call the same method RundeckApiClient._perform_request that performs the actual request.

class pyrundeck.api.RundeckApiClient(token, root_url, pem_file_path=None, client_args=None, log_level=20)[source]¶

Bases: pyrundeck.endpoints.EndpointMixins

The Rundeck API wrapper. This class is used to interact with the Rundeck server. In order to instantiate it you need to provide at least an access token and a root url for the Rundeck server.

See Usage for examples.

Parameters:
  • token – The rundeck access token.
  • root_url – The rundeck server URL.
  • pem_file_path – (optional) A file path to a CA_BUNDLE for SSL certificate validation. Default value: None.
  • client_args – (optional) Default values to be passed to every request. This should be a dictionary, notably containing a key 'headers'. Default value: None.
  • log_level – (optional) The level at which logging happens. Default value: logging.INFO.

pyrundeck.endpoints module¶

This module contains the mappings methods to the API endpoints.

Each endpoint of the API should have a corresponding method in the class EndpointMixins. The class RundeckApiClient subclasses this class in order to inherit the defined methods.

class pyrundeck.endpoints.EndpointMixins[source]¶

Bases: object

This class contains all the API endpoints in order not to clutter the pyrundeck.api.RundeckApiClient. Note that RundeckApiClient is a subclass of this class, so it inherits all the methods defined here.

The idea is to define a method for every endpoint in the Rundeck API, taking the appropriate parameters. For more details consult the Rundeck API documentation.

Warning

This class should not be instantiated and used directly. Trying to do so will definitely result in runtime errors.
