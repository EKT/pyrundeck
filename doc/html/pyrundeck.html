pyrundeck.api module¶

This module contains the core of the API client.

The general idea is to abstract away all possible requests into three methods (RundeckApiClient.get, RundeckApiClient.post and RundeckApiClient.delete), that call the same method RundeckApiClient._perform_request that performs the actual request.

class pyrundeck.api.RundeckApiClient(token, root_url, pem_file_path=None, client_args=None, log_level=20)[source]¶

Bases: pyrundeck.endpoints.EndpointMixins

The Rundeck API wrapper. This class is used to interact with the Rundeck server. In order to instantiate it you need to provide at least an access token and a root url for the Rundeck server.

See Usage for examples.

Parameters:
  • token – The rundeck access token.
  • root_url – The rundeck server URL.
  • pem_file_path – (optional) A file path to a CA_BUNDLE for SSL certificate validation. Default value: None.
  • client_args – (optional) Default values to be passed to every request. This should be a dictionary, notably containing a key 'headers'. Default value: None.
  • log_level – (optional) The level at which logging happens. Default value: logging.INFO.

pyrundeck.endpoints module¶

This module contains the mappings methods to the API endpoints.

Each endpoint of the API should have a corresponding method in the class EndpointMixins. The class RundeckApiClient subclasses this class in order to inherit the defined methods.

class pyrundeck.endpoints.EndpointMixins[source]¶

Bases: object

This class contains all the API endpoints in order not to clutter the pyrundeck.api.RundeckApiClient. Note that RundeckApiClient is a subclass of this class, so it inherits all the methods defined here.

The idea is to define a method for every endpoint in the Rundeck API, taking the appropriate parameters. For more details consult the Rundeck API documentation.

Warning

This class should not be instantiated and used directly. Trying to do so will definitely result in runtime errors.

pyrundeck.xml2native module¶

The classes and functions in this module convert the lxml.etree representation of the server response to native objects similar to a JSON representation. The functions in this module implement a recursive descent parser based on the tags of the XML representation.

The parser is driven by parsing tables that actually describe the grammar the parser accepts. In this sense this library resembles tools like yacc and bison.

class pyrundeck.xml2native.ParserEngine[source]¶

Bases: object

This is class converts the lxml.etree representation to native Python objects.

The user does not need to interact with it.

Every parse table is a dictonary containing at least one two keys: The name of the tag that we are trying to parse, specified as the string value of the 'tag' key and the type of the tag. This is specified as the value of the 'type' key inside the dict.

Currently the pyrundeck.xml2native.ParserEngine recognizes the following types of tags.

'text'

A tag that contains only text:

<text_tag>Text</text_tag>
'attribute'

A tag that contains only attributes:

<attribute_tag attrib1="foo" attrib2="bar"/>
'attribute text'

A tag that contains attributes and text:

<tag attrib="value">Text</attrib>

A special parse table dict key named 'text tag' should be specified. It’s value should be a string and it is used as the key to the returned dictionary for the text. See RundeckParser.attribute_text_tag()

'list'

A tag that is a homogeneous list of tags:

<tag>
  <child>Child 1</child>
  <child>Child 2</child>
  <child>Child 3</child>
  <child>Child 4</child>
</tag>

The value of the key 'element parse table''' should be a parse table that specifies how each element of the list will be parsed.

A special parse table dict key named 'skip count' can be set to True, to prevent the check for a count attribute in this tag.

'composite'

A tag that is composed of other tags and attributes.

The value of the key 'all'' should be a list that contains the parse tables for the mandatory child tags. The optional child tags should be specified as a list value of the 'any' tag.

'alternatives'

A tag that can be parsed in multiple ways.

The value of the key 'parse tables' should be the different parse tables that can be used to parse this tag.
